# 메타 프롬프트 생성기

상위 수준 요청 → 병렬 처리 최적화된 구조화 커스텀 슬래시 커맨드 자동 생성.

## 트리거

- 복잡한 프로젝트의 구조화된 워크플로우 요청
- 재사용 가능한 커스텀 프롬프트/슬래시 커맨드 생성 요청
- 병렬 처리 최적화가 필요한 작업 계획 요청

## 핵심 기능

1. **지식 수집**: 모델 지식 차단 이후 주제는 3개 병렬 서브 에이전트로 웹 검색
2. **단계 기반 워크플로우**: 종속성 정의, 독립 작업은 병렬/종속 작업은 순차, Task 도구 활용
3. **테스트 생성**: 단위/통합/E2E 테스트 자동 생성, 병렬 실행 여부 식별
4. **실행 전략**: 병렬 작업 자동 식별, 서브 에이전트 전략 제공

## 워크플로우

1. **컨텍스트 수집** (병렬) — 미지 주제 웹 검색, 관련 문서 수집
2. **요구사항 명확화** (순차) — 부족 정보 질문, 반복
3. **구조 설계** (순차) — 병렬화 분석, 종속성 결정, 서브 에이전트 전략
4. **콘텐츠 생성** (순차) — 지정 형식 작성, 지식 통합
5. **저장 및 보고** — `.claude/commands/<프롬프트_이름>.md`에 저장

## 생성 프롬프트 구조

```markdown
---
allowed-tools: <도구 목록>
description: <설명>
argument-hint: [<인수 힌트>]
model: sonnet
---

# <프롬프트 이름>

## 변수
- 동적 변수 ($1, $2, ...), 정적 변수

## 지침
- 실행 규칙, 엣지 케이스, 중요 요구사항
- **프로젝트별 검증**: Flutter→`flutter analyze` 에러 0개 / React→`pnpm build` 성공 / 기타→해당 정적분석+빌드

## 코드베이스 구조 (코드 관련 시)
ASCII 디렉토리 트리 + 목적 설명

## 워크플로우
### 단계 1-N: <이름>
**종속성:** <종속 단계> | **실행 모드:** 순차|병렬|혼합 | **서브 에이전트 전략:** <설명>
- 구체적 작업 내용, 조건부 로직

### 최종 단계: 검증 및 품질 보증
프로젝트별 필수 검증 실행 후 에러 0개까지 반복.

## 테스트 스위트
- 단위/통합/E2E 테스트 + 엣지 케이스
- 검증 명령: Flutter(`flutter analyze/test`), React(`pnpm build/test/lint`), 기타
- 완료 기준 체크리스트 (프로젝트별 + 공통)

## 최종 결과물 / 보고서
```

## 병렬 처리 원칙

**Task 도구 사용**: 작업 간 종속성 없음, 동시 실행 가능, 각 작업 독립적일 때
**메인 컨텍스트 실행**: 서브 에이전트 결과 통합, 순차 종속성, 상태 공유 필요 시

## 모범 사례

- 구체적 설명 제공, 점진적 개선, 버전 관리, 재사용
- 철저한 검증: Flutter→`flutter analyze` 에러 0개, React→`pnpm build` 성공

## 제한사항

- 매우 복잡한 프롬프트는 수동 조정 필요 가능
- 생성 프롬프트는 시작점, 프로젝트에 맞게 커스터마이징 권장
- 지식 차단 이후 정보는 웹 검색 의존
